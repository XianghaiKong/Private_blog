## 一、语义分割常用评价指标

在语义分割中，我们通常将预测结果和真实标签（Ground Truth）视为逐像素的分类问题。因此，评价指标往往围绕分类中的混淆矩阵展开，在多分类任务中也可以理解为二分类的推广去计算每个类别的混淆矩阵，然后对所有类别的混淆矩阵进行平均。
- **TP (True Positive)**：正确预测为正样本的像素数。
- **TN (True Negative)**：正确预测为负样本的像素数。
- **FP (False Positive)**：错误预测为正样本的像素数。
- **FN (False Negative)**：错误预测为负样本的像素数。

### 1. 像素准确率 (Pixel Accuracy, PA)
最简单的指标，指预测正确的像素占总像素的比例。*（如果在典型的二分类即区分前景/背景的场景下，可以通俗理解为下方右侧的公式；但在通用的多分类场景下，TN并无全图跨类的统一定义，应以中间的混淆矩阵主对角线累加公式为准）*
$$ PA = \frac{\sum_{i=0}^{k} p_{ii}}{\sum_{i=0}^{k} \sum_{j=0}^{k} p_{ij}} = \frac{TP + TN}{TP + TN + FP + FN} $$

### 2. 平均像素准确率 (mean Pixel Accuracy, mPA)
计算每个类别的像素准确率，然后再求所有类别的平均值。
$$ mPA = \frac{1}{k+1} \sum_{i=0}^{k} \frac{p_{ii}}{\sum_{j=0}^{k} p_{ij}} $$

### 3. 交并比 (Intersection over Union, IoU)
IoU 是语义分割中最核心常用的指标，计算某个类别的预测结果与真实标签的交集和并集的比值。对于某个特定类别：
$$ IoU = \frac{TP}{TP + FP + FN} $$

### 4. 平均交并比 (mean Intersection over Union, mIoU)
计算所有类别的 IoU，并取综合平均值。它兼顾了目标区域的精准度与完整度。
$$ mIoU = \frac{1}{k+1} \sum_{i=0}^{k} \frac{TP_i}{TP_i + FP_i + FN_i} $$

## 二、烟雾分割评价指标

烟雾（尤其是早期微小烟雾）由于其半透明、边缘模糊、画面占比极小等特性，常规的 mIoU 会在一定程度上显得颗粒度不够，还会引入类似显著性目标检测领域的更精密的判定指标：

### 1. F-measure ($F_\beta$)
F-measure 综合衡量了精确率（Precision）和召回率（Recall）。考虑到早期烟雾的检测中，我们需要平衡避免误报和防止漏报，通常会加入权重系数 $\beta^2$：
$$ F_\beta = \frac{(1 + \beta^2) \times Precision \times Recall}{\beta^2 \times Precision + Recall} $$
一般将 $\beta^2$ 设置为 $1$ ，由此计算出的 $F_\beta$ 也就是 $F_1$，如果设置 $\beta^2$ 为 $0.5$，则更侧重于精确率；如果设置 $\beta^2$ 为 $2$，则更侧重于召回率

### 2. mMse / MAE
在部分直接涉及概率图输出或者连续型 Mask 重建的网络中，**mMse (均方误差)** 或者 **MAE (平均绝对误差)** 也是很典型的指标。这两者衡量的是模型在全图逐像素置信度分布上偏离 GT 标签的细微差异：
$$ mMse = \frac{1}{W \times H} \sum_{x=1}^{W} \sum_{y=1}^{H} \left( Pred(x,y) - GT(x,y) \right)^2 $$
具有半透明度的烟雾常常在边界有渐变趋势，更低的 mMse 代表网络不仅找准了“是不是烟”，而且对“烟有多浓”的概率预判更加贴合真实场景，这里的 $Pred$ 是模型直接输出的概率图，是没有经过阈值二值化的，而 $GT$ 是真实标签，对于烟雾像素其值为 1，对于非烟雾像素其值为 0。

## 三、避坑：全局 IoU vs Image-level IoU 的致命差别

**MMSegmentation 源码中定义的指标验证，默认采用的是“全局平均精度”（Global Average）计算方式。然而在遇到类占比呈现严重长尾分布的情况（如烟雾分割、划痕检测）时，如果继续沿用该法会造假虚高，应当强制切换使用且必须理解 Image-level（单图平均）的计算机制。**

### 数学逻辑的本质差异
1. **全局平均 (Global IoU，MMSegmentation 默认)**：
   先将**整个测试集所有图片上的交集**和**所有图片上的并集**各自揉在一起累加，最后做唯一一次除法。
   $$ IoU_{global} = \frac{\sum_{n=1}^{N} 交集_n}{\sum_{n=1}^{N} 并集_n} $$

2. **单图平均 (Image-level IoU)**：
   先计算出**每一张独立图片**的内部 IoU，最后再把这 $N$ 张图的 IoU 分数加起来除以 $N$ 算术平均。
   $$ IoU_{image-level} = \frac{1}{N} \sum_{n=1}^{N} \left( \frac{交集_n}{并集_n} \right) $$

### 实例说明

假设烟雾测试集恰好只有 2 张照片，表现为典型的极端情景：
- **图片 A (大片浓烟)**：全图烟雾区域真实像素高达 10,000。模型预测交集是 $9,000$ 像素，并集是 $11,000$ 像素。单看这幅图的 $IoU_A = \frac{9000}{11000} \approx \textbf{81.8\%}$。
- **图片 B (早期微小烟雾)**：这也是极困难的情境，真实烟雾为 100 像素。然而网络预测全黑，交集为 $0$，并集为真实大小 $100$ 像素，那么$IoU_B = \frac{0}{100} = \textbf{0\%}$

如果用这两种方式结算，结果会截然不同：
*   **全局模式**：把分子分母简单合一求解：$\frac{9000 + 0}{11000 + 100} = \frac{9000}{11100} \approx \textbf{81.08\%}$
*   **单图模式**：图得分直接算术平均：$\frac{81.8\% + 0\%}{2} = \textbf{40.9\%}$

**直观结论**：当使用传统全局 IoU 时，模型在单图 B 上致命的检测失败，仅仅因为图片 B 像素绝对数量太少，被图片 A 的像素绝对数量“稀释和掩盖”了网络看着有高达 $81\%$ 的**烟雾类别单类 IoU** 幻象，采用 Image-level IoU 断崖式坠落到 $40.9\%$，更符合实际情况，适用于真实场景。

## 四、MMSegmentation 1.x 适配 Image-level 评价方法

认识到了病根，我们就需要将上述逻辑应用上。在 MMSegmentation 1.x 中更改评级指标体系非常方便，我们只需要通过覆写 `BaseMetric` 来在挂载验证时截获计算流程即可：

在你的工程环境里的 `models/metrics.py` (如果有自定义目录) 或根目录下创建一个文件并写入：

```python
from mmengine.evaluator import BaseMetric
from mmseg.registry import METRICS
import numpy as np

@METRICS.register_module()
class ImageLevelIoUMetric(BaseMetric):
    def __init__(self, num_classes, ignore_index=255, **kwargs):
        super().__init__(**kwargs)
        self.num_classes = num_classes
        self.ignore_index = ignore_index

    def process(self, data_batch, data_samples):
        """核心：对送入过程中的每一个 batch 内的每一张图独立结算 IoU 相关面积"""
        for data_sample in data_samples:
            pred_label = data_sample['pred_sem_seg']['data'].cpu().numpy().squeeze()
            label = data_sample['gt_sem_seg']['data'].cpu().numpy().squeeze()
            
            # 踢走忽略标记区（常用于图像边缘或者无需判断部分）
            mask = label != self.ignore_index
            pred_label = pred_label[mask]
            label = label[mask]
            
            # 求各块对应的面积（采用更高效、更 Pythonic 的 bincount 方法对于离散类别统计）
            intersect = pred_label[pred_label == label]
            area_intersect = np.bincount(intersect, minlength=self.num_classes)
            area_pred = np.bincount(pred_label, minlength=self.num_classes)
            area_label = np.bincount(label, minlength=self.num_classes)
            
            area_union = area_pred + area_label - area_intersect
            
            # 【关键骤：仅当前这张图交除并】
            # 使用 np.maximum 增加防分母为 0 除零断言崩溃处理 
            iou = area_intersect / np.maximum(area_union, 1e-5) 
            
            # 掩码筛选：记下当前这幅图确实含有（或错认含有）的判定类
            valid_class = area_union > 0
            
            # 将独立的这图信息送进缓存堆叠栈中
            self.results.append({'iou': iou, 'valid_class': valid_class})

    def compute_metrics(self, results):
        """结尾处再进行图级维度的统一算术处理"""
        ious = np.array([res['iou'] for res in results])
        valid_classes = np.array([res['valid_class'] for res in results])
        
        class_ious = np.zeros(self.num_classes)
        for i in range(self.num_classes):
            # 取出那些包含类所在图进行有效 mean()
            valid_mask = valid_classes[:, i]
            if valid_mask.sum() > 0:
                class_ious[i] = ious[valid_mask, i].mean()
            else:
                class_ious[i] = np.nan
        
        metrics = {
            'Image_level_mIoU': np.nanmean(class_ious) * 100
        }
        for i in range(self.num_classes):
            metrics[f'Image_level_IoU_class_{i}'] = class_ious[i] * 100
            
        return metrics
```

要想让它在模型配置文件内生效，只要确保将这段代码正确引导到 Registry 系统里之后，在 `config` 主配置内修改：

```python
# 找到你的评测设置配置域进行对位修改：
val_evaluator = dict(type='ImageLevelIoUMetric', num_classes=2, ignore_index=255)
test_evaluator = val_evaluator
```
