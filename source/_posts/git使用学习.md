## 前言

学习 Git 最怕的就是一开始就陷入各种复杂的底层原理和冗长的命令参数中。对于绝大多数日常开发而言，我们只需要掌握它作为“代码版本管理工具”的核心用法即可。

这篇博客主打**“不求搞懂全部原理，只求快速上手工作”**，整理了个人在本地代码管理及 GitHub 交互中最常用的一套极简指令集。

---

## 一、 起步：配置与初始化

在使用 Git 之前，你需要告诉 Git “你是谁”，这样后续的代码提交记录才知道是谁改的代码。

### 1. 配置个人姓名和邮箱
打开终端，运行以下命令（加上 `--global` 代表全局生效）：
```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

### 2. 获取一个代码库
获取一个受 Git 控制的代码库有两种方式：

**方式 A：从零开始初始化一个新的本地项目**
进入你的项目文件夹，执行初始化命令，这会在当前目录下生成一个隐藏的 `.git` 文件夹：
```bash
cd your_project_folder
git init
```

**方式 B：克隆别人的项目**
如果是从 GitHub 下载别人的开源项目，不需要 `git init`，直接克隆即可，它会自动包含所有版本信息：
```bash
git clone https://github.com/用户名/项目名.git
```

---

## 二、 日常开发：如何提交更改

在本地写好代码后，保存版本库通常只需要两步：“添加（Add）”和“提交（Commit）”。

### 1. 添加到暂存区
告诉 Git 你想把哪些修改了的文件纳入下一次的提交中。通常我们直接把当前目录下的所有更改都加进去：
```bash
# 添加所有更改的文件（最常用）
git add .
```

### 2. 提交到本地仓库
给这次的更改起一个备忘的名字（Commit Message），把它正式封存为一个版本：
```bash
git commit -m "新增了烟雾分割的数据处理脚本"
```
> 💡 **提示**：良好的 Commit Message 习惯非常重要，最好能简单说明这次提交做了什么。

---

## 三、 时光机：如何回到某个版本

如果代码改崩了，或者想丢弃掉最近的错误修改，Git 提供了强大的回退功能。

### 1. 查看历史版本
首先，查看以往的提交记录，找到你想回去的那个版本的“哈希值（一串黄色字符）”：
```bash
git log --oneline
```

### 2. 代码回退
通常有两种回退需求：

- **撤销还没有 commit 的修改**（让代码回到上一次提交的状态）：
  ```bash
  # 丢弃工作区所有的修改（危险操作，未保存的代码将丢失！）
  git checkout .
  # 或者更现代的命令
  git restore .
  ```

- **彻底回退到历史的某个 commit**：
  ```bash
  # 回退到指定的 commit_id，并且丢弃这之后的所有修改（极度危险，确认无误再用）
  git reset --hard <commit_id>
  ```

## 四、 平行宇宙：分支与合并

**什么时候需要新分支？**
当你正在开发一个新功能（或者修复一个 Bug），但不希望影响现有的主线稳定代码时，就需要切出一个新分支。在这个独立的分支里，你可以随便改，改坏了也没事。

### 1. 创建并切换到新分支
```bash
# 创建一个名为 dev-feature 的新分支，并立刻切换过去
git checkout -b dev-feature

# (注：较新的 Git 版本更推荐使用 switch 命令)
git switch -c dev-feature
```

### 2. 在分支上开发与提交
在新分支上，一切操作和之前一样：
```bash
git add .
git commit -m "完成了新功能开发"
```

### 3. 合并分支
功能开发完毕且测试通过后，你需要把它合并回主分支（通常叫 `main` 或 `master`）：
```bash
# 第一步：先切换回主分支
git checkout main

# 第二步：把开发分支合并进当前所在的（主）分支
git merge dev-feature
```

## 五、 云端同步：推送到 GitHub

本地的代码玩得再溜，最终也是要推送到远程（如 GitHub）备份或与他人共享的。

### 1. 关联远程仓库（仅针对 git init 初始化的项目）
如果是 `git clone` 下来的项目，它已经默认关联了远程仓库，这一步可以跳过。如果是你自己 `git init` 创建的本地项目，需要先跟 GitHub 上的空仓库建立关联：
```bash
git remote add origin https://github.com/用户名/你的仓库名.git
```

### 2. 推送本地代码
把本地的 `main` 分支推送到远程仓库的 `main` 分支（第一次推送通常需要加 `-u` 参数绑定）：
```bash
git push -u origin main
```

之后日常开发中，只要分支绑定好了，推送只需极其简单的两步走：
```bash
git commit -m "日常代码更新"
git push
```
