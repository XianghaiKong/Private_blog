RTX 5090 深度学习环境显卡掉线排查记录

发表于 2026-04-29 更新于 2026-05-31 分类于 深度学习

先交代一下这套深度学习环境的硬件配置：

  • 显卡：技嘉 RTX 5090 纯血版（国外背回来的）
  • CPU：AMD 9950X
  • 主板：华硕 X870E-PLUS WIFI7
  • 固态：三星 990 Pro 2TB
  • 机械硬盘：西部数据紫盘 2TB
  • 机箱：安钛克 FLUX SE
  • 电源：安钛克 NE 1300 金牌
  • 风冷散热：九州风神 阿萨辛4
  • 内存：美商海盗船 DDR5 5200 32G✖️2

最近在跑深度学习训练时，接连遇到几类不同表现的黑屏和 GPU 掉盘问题。起初我以为在 BIOS 里禁用核显就能解决，但实际测试下来发现问题仍然存在。为了方便日后查阅，也给遇到类似问题的朋友一个参考，这里把现象、日志证据和排查路径记录下来。

一、最初遇到的三类现象

情况一：直接掉盘，找不到 GPU 节点

这是比较棘手的一种情况。当时正在跑训练，中途桌面突然消失，机箱风扇维持高转速。通过终端 SSH 连进去后，输入 nvidia-smi 提示找不到显卡设备，表现为典型的 GPU 掉盘。

初步分析，这多半是在负载突变或者供电出现瞬时波动时，触发了主板或电源的某种保护机制，导致 PCIe 设备被系统断开。针对这种硬件级别的保护掉线，最直接的恢复方式只能是硬重启机器。

情况二：能识别显卡，但无信号输入且无桌面

这种情况表现为：开机后 nvidia-smi 能正常找到显卡并输出状态，但是显示器提示无信号输入，使用 ToDesk 远程连接时也显示没有桌面环境。

遇到这种状况，我的处理方式是彻底断电重置：关机，拔掉主机电源线，静置等待一分钟左右，然后再插上电源开机。通常这样操作一次就能恢复正常。推测可能是主板静电或某些电容的残余电荷干扰了显示输出，拔电静置可以帮助主板彻底放电。

情况三：远程有桌面，但本地显示器黑屏

第三种情况有些特殊：本地显示器依然毫无反应（无输入信号），但是通过 ToDesk 远程连接，居然可以正常看到并操作 Ubuntu 桌面。

经过摸索，这主要可能是 Ubuntu 下的显示服务器配置异常，或者显卡驱动与 Wayland 发生冲突。可以在 ToDesk 里打开终端，尝试重置相关配置：

二、第一次总结：先怀疑散热和供电

一开始我猜测是核显和独显在抢占输出通道，所以在 BIOS 里把主板核显禁用了。但事实证明，即使禁用了核显，上述黑屏和掉线问题依然会不定期出现。

综合来看，我觉得 5090 这张卡的瞬态功耗和供电要求确实非常高，即便搭配了 1300W 的金牌电源，依然会在某些特定高负载工况下出现掉线。另外，这种情况我也会更多地考虑散热方面的影响，毕竟高功耗伴随着高发热。后续我打算加装机箱风扇、定期清理灰尘，并把机箱移动到更加通风的位置，因为 1300W 电源在理论上是不可能出现余量不足的。

三、再次复现：batch 增大后频繁掉盘

后续又遇到了一次更稳定可复现的问题：batch 设为 6 时，可以轻松跑完 40k 步，此时显存只用约 10GB；将 batch 改为 12 后，显存占用约 18GB，训练跑到约 8000 步时显示器黑屏，nvidia-smi 找不到显卡。

当时 SSH 仍能连上，风扇狂转，训练实际已经停止。物理强制关机重启后一切恢复正常，但再次开跑仍会出现上述情况，这说明它已经不是偶发现象。

结合大模型辅助分析、网上资料和本机硬件配置，当时列出了几个可能原因：

  1. 散热不佳（可能性最高）

    • 主机放在桌子下面，空间狭小，通风不良。

    • 计划在机箱下方和上方加装风扇：下方进冷风，上方排热风；上方至少可加三个风扇，下方尚未查看，应该也能加装。

    • 验证命令（重点观察温度）：

      • GPU Current Temp：核心温度，一般不要超过 80°C。
      • Memory Current Temp：显存温度，最关键。若在黑屏前一路飙升到 100°C 以上甚至逼近 105°C，基本可以断定是显存过热导致掉盘。

  2. 电源余量不足（可能性较低）：满载时整机功耗过大，1300W 电源可能不够。

  3. 硬件连接（可能性较低）：显卡支架不稳、供电线插接不牢或弯折过大。已检查，暂无问题。

当时的计划是：先把主机从桌子底下挪到机架上，并加装风扇；如果之后不再出现“显存总共 32GB、刚用了 20GB 左右就桌面崩溃”的情况，就说明大概率是散热或通风问题；如果仍然不好，再考虑联系售后或上门维修。

四、再次更新：开训 14 秒就崩溃

后来我已经把机器搬到了机架上，也更换了工位。再次启动训练时，又遭遇桌面突然消失、显示器黑屏、风扇狂转，而且这次仅仅开训 14 秒就崩了。

这时基本可以确定：这不是普通的显存 OOM。OOM 通常只会让 Python 进程报错退出，不会让整机黑屏、GPU 掉盘、风扇狂转并且必须硬关机。

崩溃前内核日志出现大量 NVIDIA GSP 相关错误：

这表示 NVIDIA GSP（GPU System Processor）固件 RPC 通信失败或超时。换句话说，GPU 的 GSP 固件已经挂掉，驱动只能疯狂重试。一旦 GSP 死掉，挂在 GPU 上的 Xorg 显示就会冻结（黑屏、ToDesk 掉线），风扇也可能因为失去控制进入 100% 失效保护状态。

继续往前看崩溃第一现场，在 16:12:16，也就是启动训练那一刻，出现了更关键的日志：

其中 Xid 79 = GPU 从 PCIe 总线上掉了。这基本可以解释所有现象：GPU 整个从总线消失后，后面那一大片 GSP RPC 失败只是驱动在对一个“已经不存在的 GPU”疯狂重试的连锁反应。Node Reboot Required 也解释了为什么必须硬关机才能恢复。

另外还有几个重要细节：

  • 训练日志目录时间戳是 16:12:02，而 GPU 掉总线发生在 16:12:16，开训仅 14 秒就崩了。
  • 这正是 GPU 从待机功耗（约 54W）猛地拉载冲向高功耗的瞬间。
  • 崩溃前没有过热迹象（HW Thermal Slowdown 全为 0，且只训练了 14 秒）。
  • 没有 PCIe AER 链路报错。
  • GPU 功率上限为 600W，最低可限制到 400W。

这时的初步判断是：RTX 5090 的瞬时功耗尖峰导致供电掉电（brownout），从而让 GPU 掉出 PCIe 总线（Xid 79）。这个问题和显存、训练代码本身关系不大。

五、第一轮判断：瞬时功耗或供电保护

第一轮排查中，最符合现象的判断是：GPU 瞬时功耗尖峰导致供电掉电，GPU 掉出 PCIe 总线（Xid 79）。

证据链如下：

  1. Xid 79: GPU has fallen off the bus + Xid 154: Node Reboot Required：开训 14 秒时触发，GPU 整个从总线消失。
  2. 之后满屏的 GSP RPC failed status 0xf 只是驱动对“已消失的 GPU”疯狂重试的连锁反应，不是病因。
  3. 黑屏 / ToDesk 掉线 = 挂在该 GPU 上的 Xorg 显示瞬间冻结；风扇狂转 = GPU 失控后进入 100% 失效保护；必须硬关机 = 内核提示的 Node Reboot Required。
  4. 无过热、无 PCIe AER 链路错误，因此初步排除散热和逐渐恶化的链路报错，指向供电瞬态。

RTX 5090 标称功耗约 575–600W，但毫秒级瞬时尖峰可能冲到 900W+。如果电源额定不够，或者 12VHPWR / 12V-2x6 供电线接触不良、使用转接头，开训那一刻拉满载就可能瞬间掉电。

  • 若仍然崩溃：继续压制瞬时峰值，或者转向 PCIe 链路问题排查。

六、第二轮排查：电源容量足够后，怀疑 PCIe Gen5 链路

后续我补充了完整硬件配置，并确认电源线是原生线且插得很紧。这样一来，电源容量不足这个原因就基本可以排除了：

  • 安钛克 NE1300G 是 1300W 金牌 ATX 3.0；
  • 原生 12V-2x6 供电线；
  • 5090 + 9950X 的额定功率余量理论上足够；
  • 供电线已经重新检查，确认插紧。

这时问题更可能转向另一个方向：PCIe Gen5 链路稳定性。

排查 PCIe 链路代数时，看到当前 GPU 正运行在 PCIe Gen5 x16：

结合这套硬件配置，矛头开始指向 50 系显卡 + AM5 / X870E 平台上更容易出现的问题：PCIe Gen5 链路在满载瞬间不稳定，导致 GPU 掉出总线（Xid 79）。

原因是 RTX 50 系 + AM5 平台跑 PCIe Gen5 x16 时，信号完整性要求极高。开训那一刻，GPU 链路状态切换叠加负载骤增，Gen5 链路可能瞬间崩掉，表现就是“毫无征兆掉总线、黑屏、必须硬重启”。这和当前现象以及 Xid 79 完全吻合。

同时，我这里没有过热、没有 PCIe AER 渐进报错，电源容量也足够。因此后续重点应放在链路稳定性 / 瞬态稳定性上。

七、后续排查顺序

第 1 步：功耗上限测试

先用功耗上限测试区分病因：

  • 能稳定训练：偏向瞬态供电问题，可能是电源 OCP 对 5090 尖峰比较敏感。
  • 仍然崩溃：基本就转向 PCIe Gen5 链路问题，继续做第 2 步。

第 2 步：BIOS 中将显卡插槽降到 PCIe Gen4

这是 50 系显卡掉总线问题中比较有效的修复方式。

第 3 步：更新主板 BIOS

X870E 早期 BIOS 对 Gen5 显卡稳定性可能存在兼容性问题，华硕后续 AGESA 更新修复了不少相关问题。建议去华硕官网下载安装 X870E-PLUS WIFI7 的最新 BIOS。

八、是否需要联系售后

目前暂时不建议立刻叫维修师傅。根据硬件清单和装机情况，这更像是 BIOS / PCIe 配置层面的兼容性问题，不一定是硬件损坏。

只有在以下几项都做完后仍然崩溃，才建议考虑售后或上门检测：

  • 限功耗到 400W；
  • PCIe 从 Gen5 降到 Gen4；
  • 主板 BIOS 更新到最新；
  • 必要时关闭 ASPM。

如果这些操作都无效，才更可能是这块 5090 或这台电源存在个体瑕疵，再走保修、换件检测会更有意义。

总结

这次问题的关键并不是“显存不够”，而是 GPU 从 PCIe 总线上掉了（Xid 79）。第一轮排查指向 5090 瞬时功耗尖峰导致供电掉电；第二轮结合 1300W 原生电源线和 PCIe Gen5 x16 状态后，进一步怀疑是 PCIe Gen5 链路在满载瞬间不稳定。

后续最建议的排查顺序是：

  1. 先用 sudo nvidia-smi -pl 400 限功耗验证；
  2. 若仍然崩溃，就在 BIOS 中把显卡插槽从 Gen5 降到 Gen4；
  3. 更新主板 BIOS；
  4. 必要时再测试 pcie_aspm=off；
  5. 上述方法都无效后，再考虑售后或换件检测。

总结与后续优化

一开始我猜测是核显和独显在抢占输出通道，所以在 BIOS 里把主板核显禁用了。但事实证明，即使禁用了核显，上述黑屏和掉线问题依然会不定期出现。

综合来看，我觉得 5090 这张卡的瞬态功耗和供电要求确实非常高，即便搭配了 1300W 的金牌电源，依然会在某些特定高负载工况下出现掉线。另外，这种情况我也会更多地考虑散热方面的影响，毕竟高功耗伴随着高发热。后续我打算加装机箱风扇、定期清理灰尘，并把机箱移动到更加通风的位置，因为 1300W 电源在理论上是不可能出现余量不足的。

内容更新：再次频繁遭遇 GPU 掉盘

跑代码时，batch 设为 6 可以轻松跑完 40k 步，此时显存仅用约 10 GB；将 batch 改为 12 后，显存占用约 18 GB，训练跑到约 8000 步时显示器黑屏，nvidia-smi 找不到显卡。SSH 仍能连上，风扇狂转，训练实际已停止。物理强制关机重启后一切恢复正常，但再次开跑仍会出现上述情况——这就不是偶然了。

经过与大模型交流、网上搜索，并结合本机实际硬件配置，可能原因如下：

  1. 散热不佳（可能性最高）

    • 主机放在桌子下面，空间狭小，通风不良。
    • 计划在机箱下方和上方加装风扇：下方进冷风，上方排热风；上方至少可加三个风扇，下方尚未查看，应该也能加装。
    • 验证命令（重点观察温度）：
      • GPU Current Temp：核心温度，一般不要超过 80°C。
      • Memory Current Temp：显存温度，最关键。若在黑屏前一路飙升到 100°C 以上甚至逼近 105°C，基本可以断定是显存过热导致掉盘。

  2. 电源余量不足（可能性较低）：满载时整机功耗过大，1300 W 电源可能不够。

  3. 硬件连接（可能性较低）：显卡支架不稳、供电线插接不牢或弯折过大——已检查，暂无问题。

总结就是，后续会将主机挪到机架上，也就是不在桌子底下，并加装风扇，到时候再跑一下代码尝试，如果不再出现上述情况（比如显存总共32G，而刚用了20G就桌面崩溃），那就解决了，如果还没好，那么就从京东喊师傅上门维修。

再次更新：
此时已经把机器搬到了机架上，也更换了工位，再次想跑训练代码时，又遭遇桌面突然消失，显示器黑屏，风扇狂转，而且这次仅仅开训14秒就这样了，在cursor中问了大模型，结论如下：

这是系统级硬崩溃（黑屏 + 掉线 + 风扇狂转 + 必须硬关机），不是普通的显存 OOM —— OOM 只会让 Python 进程报错退出，绝不会让整机断电。最可能是电源(PSU)瞬时功率不足 / 驱动崩溃 / 供电保护触发。5090 瞬时功耗尖峰极高，这是经典症状。

关键证据已经出现。崩溃前内核日志被刷屏：

NVRM: _issueRpcAndWait: rpcSendMessage failed with status 0x0000000f for fn 76
这是 NVIDIA GSP（GPU 系统处理器）固件 RPC 通信失败/超时 —— GPU 的 GSP 固件挂掉了，驱动在疯狂重试。一旦 GSP 死掉，挂在 GPU 上的 Xorg 显示就冻结（黑屏、ToDesk 掉线），风扇因失去控制进入 100% 失效保护（狂转）。这是驱动/GSP 固件问题，不是显存不够。

我要看清崩溃的第一现场：RPC 刷屏之前发生了什么（是否有 Xid 错误、电源、温度告警）。

找到决定性证据了。崩溃第一现场（16:12:16，就在你启动训练那一刻）：

NVRM: Xid (PCI:0000:01:00): 79, … GPU has fallen off the bus.
NVRM: GPU 0000:01:00.0: GPU has fallen off the bus.
NVRM: Xid 154, GPU recovery action … Node Reboot Required
Xid 79 = “GPU 从 PCIe 总线上掉了” —— 这是教科书级的硬件/供电故障signature。GPU 整个从总线消失后，后面那一大片 GSP RPC 失败只是驱动在对一个"已经不存在的 GPU"疯狂重试的连锁反应。Node Reboot Required 也解释了为什么你必须硬关机才能恢复。

让我再取证：确认这是反复发生、并排查是否伴随 PCIe 链路错误。

诊断已经完全清晰了。再补两个决定性细节：

训练日志目录时间戳 16:12:02，而 GPU 掉总线发生在 16:12:16 —— 开训仅 14 秒就崩了。这正是 GPU 从待机（~54W）猛地拉载冲向高功耗的瞬间。
崩溃前无任何过热（HW Thermal Slowdown 全 0，崩时才训 14 秒）、无 PCIe AER 链路报错。属于"毫无征兆、瞬间掉总线"。
GPU 功率上限 600W，最低可限到 400W。
这是 RTX 5090 经典的瞬时功耗尖峰导致供电掉电（brownout）→ GPU 掉出 PCIe 总线（Xid 79） 的典型案例，跟显存、跟你的代码完全无关

一、根本原因（已确诊）

GPU 瞬时功耗尖峰导致供电掉电，GPU 掉出 PCIe 总线（Xid 79）。 与显存、与你的训练代码无关。

证据链：

  1. Xid 79: GPU has fallen off the bus + Xid 154: Node Reboot Required —— 开训 14 秒时触发，GPU 整个从总线消失。
  2. 之后满屏的 GSP RPC failed status 0xf 只是驱动对"已消失的 GPU"疯狂重试的连锁反应（不是病因）。
  3. 黑屏/ToDesk 掉线 = 挂在该 GPU 上的 Xorg 显示瞬间冻结；风扇狂转 = GPU 失控后进入 100% 失效保护；必须硬关机 = 内核提示的 “Node Reboot Required”。
  4. 无过热、无 PCIe AER 链路错误 → 排除散热、排除信号链路老化，指向供电瞬态。

RTX 5090 标称 ~575–600W，但毫秒级瞬时尖峰可冲到 900W+。如果电源额定不够、或 12VHPWR/12V-2x6 供电线接触不良/用了转接头，开训那一刻拉满载就会瞬间掉电。

三、真正的根治（硬件，按优先级）

  1. 检查 12VHPWR / 12V-2x6 供电线（最高嫌疑）：
    • 必须用电源原生那根 16-pin 线，不要用一拖二/菊花链转接头；
    • GPU 端和电源端都重新插拔到底（听到/感到卡扣到位），这是 4090/5090 掉总线最常见的元凶。
  2. 电源额定功率：5090 建议 ≥1000W（最好 1200W）ATX 3.0/3.1 且原生 12V-2x6 接口。若你现在是 850W 或更低、或老电源用转接头，几乎必然瞬态掉电。
  3. 重插 GPU 到 PCIe 槽，确认供电稳固。
  4. BIOS / 主板供电相关项更新。

四、补充建议

  • 驱动是 590.48.01（开源内核模块 + GSP），属较新版本。Xid 79 是硬件级"掉总线"，首要是供电而非驱动；但若硬件排查后限功耗仍偶发，可考虑换一个稳定驱动分支测试。
  • 日志已持久化（能读到上次崩溃记录），以后再崩可直接 journalctl -b -1 -k | grep Xid 复查。

结论：不是"显存不够"，而是 5090 瞬时功耗把供电拉崩、GPU 掉出总线。先执行第二节的 nvidia-smi -pl 400 验证，然后重点查第三节的 12VHPWR 供电线与电源额定，再考虑是否要把功率限制设置为系统开机服务，不用每次手动设置。

在我进一步与大模型沟通后，我发了硬件配置给他，并说明电源线我检查过了，插得很紧，同时也问他需不需要叫维修师傅，它进行了第二轮排查：

你这套配置非常硬核 —— 安钛克 NE1300G 是 1300W 金牌 ATX3.0、原生 12V-2x6，带 5090+9950X 容量绰绰有余，加上你确认线是原生且插紧。所以"电源功率不够"基本可以排除。那问题就更可能在另外两个方向，我先取证再下结论。

让我查 PCIe 链路代数和内核启动参数。

查到了第二个关键证据：

pcie.link.gen.current = 5 ← GPU 正跑在 PCIe Gen5 x16
结合你这套硬件（1300W 金牌原生线、线插紧），"电源容量不足"基本排除，矛头强烈指向另一个 50 系显卡 + AM5 平台的高发问题：PCIe Gen5 链路在满载瞬间不稳定，导致 GPU 掉出总线（Xid 79）。

为什么是这个原因
RTX 50 系 + AM5（X870E）跑 PCIe Gen5 x16 时，信号完整性要求极高。开训那一刻 GPU 链路状态切换 + 负载骤增，Gen5 链路容易瞬间崩掉，表现就是"毫无征兆掉总线、黑屏、要硬重启"——和你的现象、和 Xid 79 完全吻合。
你没有任何过热、没有 PCIe AER 渐进报错、电源又足够 → 不是散热、不是容量、不是线材老化，就是链路/瞬态层面的稳定性。

第 1 步：功耗上限测试（5 分钟，区分病因）

能稳定训练 → 偏向瞬态供电（电源 OCP 对 5090 尖峰敏感）。
仍然崩 → 基本就是 PCIe Gen5 链路问题，做第 2 步。
第 2 步：BIOS 里把显卡插槽降到 PCIe Gen4（这是 50 系掉总线最有效的修复）

重启按 Del 进 ASUS BIOS → Advanced（高级） → 找 PCI Subsystem Settings 或 PCIe Link Speed / PCIEX16_1 Link Speed；
把它从 Auto/Gen5 改成 Gen4 → 保存重启。
对你的训练性能几乎无损（Gen4 x16 带宽对单卡训练完全够），但稳定性大幅提升。
第 3 步：更新主板 BIOS

X870E 早期 BIOS 对 Gen5 显卡稳定性有已知问题，华硕后续 AGESA 更新修了很多。到华硕官网下 X870E-PLUS WIFI7 最新 BIOS 刷一下。
第 4 步（可选）：关闭 ASPM

内核启动参数加 pcie_aspm=off（我可以帮你改 GRUB），排除省电状态切换导致掉链路。
