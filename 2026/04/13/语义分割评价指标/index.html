语义分割评价指标

发表于 2026-04-13 更新于 2026-04-14 分类于 深度学习

在进行语义分割任务（尤其是像烟雾分割这种带有类极度不平衡特点的任务）时，选择和理解正确的评价指标至关重要。本文将从语义分割的常用指标讲起，然后结合我师兄针对早期烟雾分割提出的 FoSp 论文及其代码库，详细探讨烟雾分割中常用的高级评价指标。最后，我们将重点分析默认情况下 MMSegmentation 中的全局 IoU 与 Image-level（单图平均）IoU 的本质区别，并给出在 MMSegmentation 1.x 版本中的代码实现。

一、语义分割常用评价指标

在语义分割中，我们通常将预测结果和真实标签（Ground Truth）视为逐像素的分类问题。因此，评价指标往往围绕分类中的混淆矩阵展开：

  • TP (True Positive)：正确预测为正样本的像素数。
  • TN (True Negative)：正确预测为负样本的像素数。
  • FP (False Positive)：错误预测为正样本的像素数。
  • FN (False Negative)：错误预测为负样本的像素数。

1. 像素准确率 (Pixel Accuracy, PA)

最简单的指标，指预测正确的像素占总像素的比例。（注：如果在典型的二分类即区分前景/背景的场景下，可以通俗理解为下方右侧的公式；但在通用的多分类场景下，TN并无全图跨类的统一定义，应以中间的混淆矩阵主对角线累加公式为准）

PA=∑i=0kpii∑i=0k∑j=0kpij=TP+TNTP+TN+FP+FNPA = \frac{\sum_{i=0}^{k} p_{ii}}{\sum_{i=0}^{k} \sum_{j=0}^{k} p_{ij}} = \frac{TP + TN}{TP + TN + FP + FN} PA=∑i=0k​∑j=0k​pij​∑i=0k​pii​​=TP+TN+FP+FNTP+TN​

2. 平均像素准确率 (mean Pixel Accuracy, mPA)

计算每个类别的像素准确率，然后再求所有类别的平均值。

mPA=1k+1∑i=0kpii∑j=0kpijmPA = \frac{1}{k+1} \sum_{i=0}^{k} \frac{p_{ii}}{\sum_{j=0}^{k} p_{ij}} mPA=k+11​i=0∑k​∑j=0k​pij​pii​​

3. 交并比 (Intersection over Union, IoU)

IoU 是语义分割中最核心常用的指标，计算某个类别的预测结果与真实标签的交集和并集的比值。对于某个特定类别：

IoU=TPTP+FP+FNIoU = \frac{TP}{TP + FP + FN} IoU=TP+FP+FNTP​

4. 平均交并比 (mean Intersection over Union, mIoU)

计算所有类别的 IoU，并取综合平均值。它兼顾了目标区域的精准度与完整度。

mIoU=1k+1∑i=0kTPiTPi+FPi+FNimIoU = \frac{1}{k+1} \sum_{i=0}^{k} \frac{TP_i}{TP_i + FP_i + FN_i} mIoU=k+11​i=0∑k​TPi​+FPi​+FNi​TPi​​

二、烟雾分割增强指标（基于师兄 FoSp 论文体系）

烟雾（尤其是早期微小烟雾）由于其半透明、边缘模糊、画面占比极小等特性，常规的 mIoU 会在一定程度上显得颗粒度不够。在我师兄提出的针对早期烟雾分割的 FoSp (Focus and Separation Network) 等前沿论文代码库中，往往引入类似显著性目标检测领域的更精密的判定指标：

1. F-measure (FβF_\betaFβ​)

F-measure 综合衡量了精确率（Precision）和召回率（Recall）。考虑到早期烟雾的检测中，我们需要平衡避免误报和防止漏报（往往倾向于突出精准率），通常会加入权重系数 β2\beta^2β2：

Fβ=(1+β2)×Precision×Recallβ2×Precision+RecallF_\beta = \frac{(1 + \beta^2) \times Precision \times Recall}{\beta^2 \times Precision + Recall} Fβ​=β2×Precision+Recall(1+β2)×Precision×Recall​

在显著性检测和烟雾分割约定俗成的习惯中，常将 β2\beta^2β2 设置为 0.30.30.3 ，由此计算出的 FβF_\betaFβ​ 能更好地评价烟雾主体的轮廓清晰度。

2. mMse / MAE

在部分直接涉及概率图输出或者连续型 Mask 重建的网络中，mMse (均方误差) 或者 MAE (平均绝对误差) 也是很典型的指标。这两者衡量的是模型在全图逐像素置信度分布上偏离 GT 标签的细微差异：

mMse=1W×H∑x=1W∑y=1H(Pred(x,y)−GT(x,y))2mMse = \frac{1}{W \times H} \sum_{x=1}^{W} \sum_{y=1}^{H} \left( Pred(x,y) - GT(x,y) \right)^2 mMse=W×H1​x=1∑W​y=1∑H​(Pred(x,y)−GT(x,y))2

具有半透明度的烟雾常常在边界有渐变趋势，更低的 mMse 代表网络不仅找准了“是不是烟”，而且对“烟有多浓”的概率预判更加贴合真实场景。

三、避坑：全局 IoU vs Image-level IoU 的致命差别

这是一个在套用通用开源库打比赛或做科研时最容易死板踩坑的地方！MMSegmentation 源码中定义的指标验证，默认采用的是“全局平均精度”（Global Average）计算方式。然而在遇到类占比呈现严重长尾分布的情况（如烟雾分割、划痕检测）时，如果继续沿用该法会造假虚高，应当强制切换使用且必须理解 Image-level（单图平均）的计算机制。

数学逻辑的本质差异

  1. 全局平均 (Global IoU，MMSegmentation 默认)：
    先将整个验证集乃至测试集所有图片上的交集和所有图片上的并集各自揉在一起累加，最后做唯一一次除法。

    IoUglobal=∑n=1N交集n∑n=1N并集nIoU_{global} = \frac{\sum_{n=1}^{N} 交集_n}{\sum_{n=1}^{N} 并集_n} IoUglobal​=∑n=1N​并集n​∑n=1N​交集n​​

  2. 单图平均 (Image-level IoU)：
    先行计算出每一张独立图片的内部 IoU，最后再把这 NNN 张图的 IoU 分数加起来除以 NNN 算术平均。

    IoUimage−level=1N∑n=1N(交集n并集n)IoU_{image-level} = \frac{1}{N} \sum_{n=1}^{N} \left( \frac{交集_n}{并集_n} \right) IoUimage−level​=N1​n=1∑N​(并集n​交集n​​)

举例硬核解析：“为什么我的网络预测烂透了但 mIoU 依然很高？”

假设我们的烟雾测试集恰好只有 2 张照片，表现为典型的极端情景：

  • 图片 A (厂房火灾大片浓烟)：全图烟雾区域真实像素高达 10,000。你的模型碰巧捕捉得很准，预测交集是 9,0009,0009,000 像素，并集是 11,00011,00011,000 像素。单看这幅图的 IoUA=900011000≈81.8%IoU_A = \frac{9000}{11000} \approx \textbf{81.8\%}IoUA​=110009000​≈81.8%。
  • 图片 B (早期初萌微小烟雾)：这也是极困难的情境，真实烟雾才寥寥 100 像素大。结果网络退化严重直接看走眼漏掉了，预测全黑，交集为 000，并集为真实大小 100100100 像素。单幅这图的成绩惨不忍睹，IoUB=0100=0%IoU_B = \frac{0}{100} = \textbf{0\%}IoUB​=1000​=0%。

我们看看如果用这两种方式结算，结果会截然不同：

  • 全局模式：把分子分母简单合一求解：9000+011000+100=900011100≈81.08%\frac{9000 + 0}{11000 + 100} = \frac{9000}{11100} \approx \textbf{81.08\%}11000+1009000+0​=111009000​≈81.08%
  • 单图模式：图得分直接算术平均：81.8%+0%2=40.9%\frac{81.8\% + 0\%}{2} = \textbf{40.9\%}281.8%+0%​=40.9%

直观结论：当使用传统全局 IoU 时，模型在单图 B 上致命的检测失败，仅仅因为图片 B 像素绝对数量太少，被图片 A 庞然巨物的惊人绝对数量无情“稀释和掩盖”了！！！网络看着有高达 81%81\%81% 的烟雾类别单类 IoU 幻象，但如果上到实战中检测微弱早期烟雾，结果必死无疑。而采用 Image-level IoU 断崖式坠落到 40.9%40.9\%40.9%，才是扯下了遮羞布，真正客观反映出了模型并没有解决早期烟雾识别难题。
